Пошаговый план разработки для ИИ-агента
=======================================

Цветочный календарь Беларуси
============================

Главный принцип работы агента
-----------------------------

Агент должен работать так:

1.  Делать один маленький шаг за раз

2.  После каждого шага:

-   запускать проект;

-   проверять, что ничего не сломано;

-   фиксировать результат;

4.  Не переходить к следующему шагу, пока текущий не завершен

5.  Не рефакторить лишнее без необходимости

6.  Сначала делать рабочий MVP, потом улучшать

* * * * *

Общие правила для агента
========================

Что агенту можно
----------------

-   создавать файлы и папки;

-   настраивать проект;

-   писать компоненты;

-   писать типы;

-   писать простые route handlers;

-   писать SQLite-слой;

-   добавлять тестовые данные;

-   подключать UI-библиотеки;

-   делать постепенные коммиты.

Что агенту нельзя
-----------------

-   сразу делать "идеальную архитектуру" на 100 файлов;

-   добавлять лишние библиотеки без необходимости;

-   смешивать UI, данные и бизнес-логику в одном компоненте;

-   переходить к сложным страницам до завершения главной;

-   трогать готовый рабочий код без причины;

-   делать огромные рефакторы между шагами.

* * * * *

Целевой MVP
===========

К концу MVP должно быть:

-   главная страница;

-   логотип;

-   заголовок;

-   переключатель режимов Города / Растения;

-   autocomplete для первого поля;

-   выбор диапазона дат;

-   кнопка поиска;

-   блок "Сейчас цветет в Минске";

-   результаты поиска карточками;

-   отдельная страница растения;

-   отдельная страница города;

-   данные из SQLite;

-   длинные тексты из markdown/mdx;

-   изображения из /public.

* * * * *

Этап 0. Правила выполнения задач агентом
========================================

Перед началом разработки агент должен принять следующие правила:

Формат каждой итерации
----------------------

Для каждого шага агент должен:

1.  кратко написать, что делает;

2.  изменить только нужные файлы;

3.  после изменений:

-   выполнить install/build/lint;

-   проверить отсутствие ошибок;

5.  коротко отчитаться:

-   что сделано;

-   что готово;

-   что следующий шаг.

Definition of Done для каждого шага
-----------------------------------

Шаг завершен, если:

-   код компилируется;

-   нет явных runtime ошибок;

-   нет TypeScript ошибок;

-   интерфейс или логика реально работают;

-   шаг не ломает предыдущий функционал.
